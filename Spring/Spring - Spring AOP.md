# Spring - Spring AOP

## 0. 용어 검색

```
1. AOP, Aspect Oriented Programming 관점 지향 프로그래밍 : 어떤 로직을 기준으로 핵심적인 관점과 부가적인 관점으로 나누어서 보고 각 관점을 기준으로 각각 모듈화(모듈화 : 어떤 공통된 로직이나 기능을 하나의 단위로 묶는 것)한다. 
2. target of AOP : Aspect를 적용하는 곳.
3. Adive  of AOP : 실질적인 부가기능을 담은 구현체. 
4. PointCut of AOP : JoinPoint의 상세한 스펙을 정의해 Advice가 더욱 구체적으로 실행될 지점을 정할 수 있게 된다.
5. JoinPoin of AOP : Adivce가 적용될 위치. 끼어들 수 있는 지점, 메서드 진입 지점, 생성자 호출 시점, 필드에서 값을 꺼내올 때 등 다양한 시점에 적용이 가능하다.
6. Aspect (= Adive + PointCut) : 흩어진 관심사(소스 코드상 부분을 계속 반복해서 쓰는 코드)를 모듈화 한것. 주로 부가기능을 모듈화 한다.
7. Adivsor
8. Weaving : 횡단 관심 메서드가 삽입되는 과정. 해당 과정을 통해 비즈니스 메소드를 수정하지 않고 횡단 관심에 해당하는 기능을 추가하거나 변경이 가능해진다.
```
