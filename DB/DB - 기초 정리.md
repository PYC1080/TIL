## 1. DB 설계 기초지식

```
1. DB 설계 시 3가지 고민
	1) 엔티티 도출 방ㅇ법
	2) 기본 키 설정
	3) 중복 제거(정규화)
2. 엔티티 도출 방법
	1) 엔티티 : 무엇인가의 집합
	2) 6W3H에 의한 엔티티명 분류
		1) 엔티티 후보
        	a) 리소스 계열
        		(a) Whom : 누구에게
        		(b) Who : 누가
        		(c) What : 무엇을 
        		(d) Where : 어디서
        	b) 이벤트 계열
        		(a) How : 어떻게
        2) 속성 후보
        	a) When : 언제
        	b) How many : 어느정도
        	c) How much : 얼마나
        3) 정규화 대상
        	a) why : 왜
3. 키 설정 방법
	1) 키 : 어떠한 데이터를 찾아내고 존재하는 데이터에 도달하기 위한 단서
	2) 키의 종류
		(1) 외래 키(참조 키) :엔티티 사이에 걸친 인스턴스 사이의 관계를 표현
		(2) 기본 키 : NOT NULL의 고유 인덱스. 인스턴스를 검색 가능한 인식자
		(3) 후보 키 : 인덱스. 엔티티 안의 인스턴스를 각각 검색 가능한 모든 인식자.
		(4) 고유 키 : 고유 인덱스. 엔티티 안의 인스턴스를 각각 검색 가능한 인식자.
4. 업무 시스템의 정규화
	1) 정규화 : 최소한의 데이터로 필요한 충분한 결과를 만들어 낸다. 즉, 정규하는 중복의 제거이다.
	2) 정규화 고려요소
		(1) 가역성 : 어떠한 상태로 변하였다가 본래 상태로 되돌아갈 수 있는 성질.
		(2) 가외성의 배제 : 실제 필요한 데이터 양보다 많은 데이터 양을 보유하는 것을 배제
5. DB 설계 순서
	1) 대략적인 블록 분류 시행
	2) 각각의 블록마다 이벤트 계열 엔티티 도출. 해당 시점에서는 정규화를 고려하지 않음
	3) 이벤트 계열 엔티티에 정규화를 하고 리소스 게열 엔티티를 도출
	4) 리소스 계열 엔티티의 분류를 도출하고 리소스 계열 엔티티의 정규화 수행
	5) 블록 상이에서 리소스 계열 엔티티의 통합을 수행하고 정규화를 수행
	6. 도출 게열을 정리하고 최종적인 정규화를 수행    		
```
